# 0001: Use Node.js for bootstrap and upkeep logic

We use Node.js because the owner already runs a Volta-pinned Node toolchain and Nightshift uses the same ecosystem. Bash would have a lower bootstrap barrier, but the subprocess-heavy install logic needs mockable tests and clearer error handling than shell scripts usually provide. Rust would give stronger compile-time guarantees, but it adds build complexity and slower iteration for laptop-maintenance scripts that mostly orchestrate existing macOS tools. Python would also be reasonable, but Node's built-in `node:test` runner lets the repo stay dependency-light while matching the owner's existing development workflow. launchd remains the scheduler because it is the macOS-native lifecycle mechanism for unattended daily jobs.
